🎓 University
Exam Instructions
Time: 3 Hours  |  Maximum Marks: 80  |  All questions are compulsory unless stated.  |  Attempt any 5 out of 8 questions in Section B.  |  Use of calculator not permitted.

Section A — Short Answer Questions [2 × 10 = 20 Marks]

Answer all 10 questions. Each question carries 2 marks.

  1. Define an operating system. State any two functions of an OS.
  2. What is the difference between preemptive and non-preemptive scheduling?
  3. Define a critical section. State any two requirements for its solution.
  4. What are the four Coffman conditions necessary for deadlock?
  5. Differentiate between internal and external fragmentation.
  6. What is a TLB (Translation Lookaside Buffer)? State its purpose.
  7. Define Turnaround Time (TAT) and Waiting Time (WT) in CPU scheduling.
  8. Compare contiguous and linked file allocation methods.
  9. What is RAID? State the difference between RAID 0 and RAID 1.
  10. What is a Virtual File System (VFS)? Why is it needed?

Section B — Long Answer Questions [10 × 6 = 60 Marks] (Attempt any 5)

Each question carries 10 marks.

Q1. (a) Explain the five-state process model with state transition diagram. [5]   (b) What is context switching? Explain what is saved/restored during a context switch. [5]

(a) Five-State Model: New → Ready → Running → Waiting → Terminated. Transitions: New→Ready (process admitted/created), Ready→Running (CPU scheduler dispatches), Running→Ready (preemption or time quantum expiry), Running→Waiting (I/O request or event wait), Waiting→Ready (I/O completion or event), Running→Terminated (process exits). Key: no direct Waiting→Running — must pass through Ready queue.

(b) Context Switch: The OS saves the state of the currently running process into its PCB (Program Counter, all CPU registers, stack pointer, memory management info, open file table pointers) and loads the saved state of the next process from its PCB. Time taken = context switch overhead = pure overhead (no useful work done). Modern CPUs minimize this with hardware support (e.g., x86 TSS, ARM register banking). Context switch required during preemption, system calls, and I/O blocking.

Q2. Four processes arrive as below. Compute average TAT and WT for: (a) FCFS (b) SJF (non-preemptive) (c) Round Robin (Q=3). Draw Gantt charts. [10]
P1(AT=0,BT=8), P2(AT=1,BT=4), P3(AT=2,BT=9), P4(AT=3,BT=5)

Q3. (a) Explain Peterson's solution to the critical section problem. Prove it satisfies mutual exclusion and bounded waiting. [6]   (b) What is priority inversion? How does priority inheritance solve it? [4]

(a) See Unit 2B → Peterson's solution. For mutual exclusion proof: Assume both P0 and P1 are in CS. Then flag[0]=flag[1]=true. P0 in CS means: either flag[1]=false (contradiction) or turn=0 (so P1 was waiting). But P1 also in CS means turn=1. Contradiction — only one can be in CS. Bounded waiting: if Pi wants CS, it sets turn=j. Then Pj enters CS at most once before Pi gets CS (turn changes). ✅

(b) Priority inversion: High-priority H blocks waiting for resource held by Low-priority L. Medium-priority M preempts L, running before H effectively. Fix — Priority Inheritance: When H blocks on L's mutex, L temporarily gets H's priority. M cannot preempt L. L finishes, releases resource, H runs. Famous example: Mars Pathfinder (1997) reset loops until priority inheritance was enabled remotely.

Q4. Explain Banker's Algorithm for deadlock avoidance. Given: 5 processes, 3 resources (A=10, B=5, C=7). With allocation and maximum matrices shown, determine the safe sequence and whether a new request by P1 for (1,0,2) can be granted. [10]
Banker's Algorithm Solution
         Alloc      Max        Need         
         A  B  C    A  B  C    A  B  C    Available = (3,3,2)
P0       0  1  0    7  5  3    7  4  3

Q5. (a) Explain paging with a neat diagram. How does a logical address get translated to a physical address? [6]   (b) EAT with TLB: hit rate 80%, TLB time 5ns, memory time 100ns. Find EAT. [4]

(a) Paging: Logical address space divided into equal-sized pages. Physical memory divided into frames of same size. Page Table maps page numbers to frame numbers. Logical address = (page number p, offset d). Physical address = (frame number f, offset d) where f = PageTable[p]. MMU (hardware) does translation. PTBR register points to page table base in RAM. TLB (cache) holds recent p→f mappings for fast lookup.

(b) EAT: α=0.80, t=5ns, m=100ns. EAT = α(t+m) + (1-α)(t+2m) = 0.8(105) + 0.2(205) = 84 + 41 = 125 ns. Without TLB: 2×100 = 200ns. Speedup = 200/125 = 1.6×.

Q6. (a) Compare FIFO, LRU, and Optimal page replacement algorithms. Which one is best and why can't it be implemented in practice? [5]   (b) What is thrashing? How does the OS detect and handle it? [5]

(a) Optimal: Replace page not used for longest future time → minimum page faults, theoretical lower bound. Cannot be implemented because future references are unknown at runtime. LRU: Replace least recently used page — good approximation of Optimal (locality of reference). Hardware-expensive to implement exactly; approximated with clock/aging bits. FIFO: Replace oldest loaded page — simple but poor performance, suffers Bélády's anomaly (more frames → more faults possible).

(b) Thrashing: CPU utilization drops sharply even though degree of multiprogramming is high. Each process's frames are insufficient for its working set → constant page faults. Detection: monitor page fault rate vs CPU utilization. If page fault rate rises and CPU utilization falls, thrashing. Handle: (1) Working Set Model — ensure each process gets enough frames for its working set. (2) Page Fault Frequency (PFF) — if fault rate too high, give more frames; if too low, reclaim frames. (3) Suspend some processes to reduce multiprogramming.

Q7. (a) Explain disk scheduling with SSTF and SCAN for the following queue: 82, 170, 43, 140, 24, 16, 190. Head at 50, max cylinder 199. [7]   (b) What is RAID 5? How many 250GB disks are needed to get 1TB usable space with RAID 5? [3]
Disk Scheduling + RAID Solution
(a) Queue: 82,170,43,140,24,16,190 | Head=50 | Max=199

SSTF (always pick nearest to current head):
50→43(7)→24(19)→16(8)→82(66)→140(58)→170(30)→190(20)
Total = 7+19+8+66+58+30+20 = 208 cylinders

SCAN (moving right first toward 199):
Sort: 16,24,43,82,140,170,190 | Right of 50: 82,140,170,190 | Left: 43,24,16
50→82(32)→140(58)→170(30)→190(20)→199(9)→43(156)→24(19)→16(8)
Total = 32+58+30+20+9+156+19+8 = 332 cylinders

(b) RAID 5 usable = (N-1) × disk_size
1000 GB = (N-1) × 250 → N-1 = 4 → N = 5 disks (total 1250 GB, 1000 GB usable)
1 disk (250 GB) used for distributed parity. Survives 1 disk failure.
To survive 2 failures: RAID 6 → N-2=4 → N=6 disks needed.
Q8. (a) Explain indexed file allocation with the Unix inode structure. Calculate max file size: block=4KB, pointer=4 bytes, 12 direct + 1 single + 1 double indirect. [6]   (b) Explain journaling in file systems. How does it improve crash recovery? [4]

(a) Unix Inode: Index node stores all metadata + 15 block pointers: 12 direct (point to data blocks), 1 single indirect (points to index block), 1 double indirect (index→index→data), 1 triple indirect. Pointers per block = 4096/4 = 1024. Direct: 12 × 4KB = 48 KB. Single indirect: 1024 × 4KB = 4 MB. Double indirect: 1024² × 4KB = 4 GB. Triple indirect: 1024³ × 4KB = 4 TB. Max file size ≈ 4 TB (triple dominates).

(b) Journaling: Before writing actual file system data, write a log entry (journal) describing what is about to be done. After crash, OS reads journal and either replays incomplete operations (redo) or cancels them (undo). This ensures the FS is always consistent — recovery takes seconds (just replay journal) vs minutes (fsck full scan). Modes: data=ordered (default ext4 — metadata journaled, data written before metadata), data=journal (full journaling, safest but slowest).

⚡ Paper 2 — GATE-Style Mock Exam

⚡ GATE
Exam Instructions
Time: 3 Hours  |  Total Questions: 30 OS MCQs (OS-only practice, not full GATE paper)  |  GATE MCQ marking: 1-mark MCQ wrong = −1/3 mark. 2-mark MCQ wrong = −2/3 mark. MSQ and NAT: no negative marking.  |  Do NOT look at answers before attempting each question.
⏱️
Strategy: Attempt all 1-mark questions first (1 min each). Then 2-mark questions (2-3 min each). Flag uncertain ones and return. Don't leave 2-mark questions blank if you can eliminate 2 options.

Part I — 1 Mark Questions

1. Which of the following scheduling algorithms can cause starvation?
A. Round Robin   B. FCFS   C. SJF   D. Both B and C
Answer
C. SJF (and by extension SRTF). Long processes can wait indefinitely if short processes keep arriving. Priority scheduling can also starve low-priority jobs if aging is absent. FCFS: no starvation for jobs already in queue (FIFO order). RR: prevents CPU starvation under a finite ready queue and fair implementation. SJF: long job starves if short jobs keep arriving — aging solves it.
2. A semaphore initialized to 1 is used to protect a critical section. What type of semaphore is this?
A. Counting semaphore   B. Binary semaphore   C. Mutex   D. Both B and C
Answer
B. Binary semaphore. Value is only 0 or 1. It behaves like a mutex for mutual exclusion but lacks ownership semantics (any process can call signal(), not just the one that called wait()). A mutex is similar to a binary semaphore but not identical — a true mutex has ownership: only the thread/process that locked it can unlock it.
3. In a system with 5 processes and 12 identical resources, if each process can request at most 3 resources, can deadlock occur?
A. Yes   B. No   C. Only if circular wait occurs   D. Cannot be determined
Answer
B. No. Standard deadlock-freedom formula for n processes each needing at most k identical resources: m ≥ n×(k−1) + 1. Here: n=5, k=3, required = 5×(3−1)+1 = 11. Given m=12 ≥ 11 → deadlock cannot occur. Intuition: worst case each holds k−1=2 resources (total=10), 12−10=2 free → at least one process can complete.
4. For a 32-bit logical address with 12-bit page offset, the number of entries in a single-level page table is:
A. 2²⁰   B. 2¹²   C. 2³²   D. 2⁸
Answer
A. 2²⁰. Page number bits = 32−12 = 20. Number of pages (= page table entries) = 2²⁰ = 1M entries.
5. Which file allocation method is used in FAT32 file system?
A. Contiguous   B. Indexed   C. Linked   D. Hybrid
Answer
C. Linked (with FAT table in RAM). FAT32 is a linked allocation system where the linked list pointers are stored in a dedicated File Allocation Table (kept in RAM) rather than in each data block. Each FAT entry contains the next cluster number for that file.
6. Peterson's solution works correctly for:
A. Any number of processes   B. Exactly 2 processes   C. Exactly 3 processes   D. Any power of 2 processes
Answer
B. Exactly 2 processes. The original Peterson's algorithm with flag[2] and turn variable is designed for the 2-process critical section problem only. Generalizing to N processes requires a different algorithm (e.g., Lamport's Bakery Algorithm).
7. Which page replacement algorithm gives the minimum number of page faults?
A. LRU   B. FIFO   C. Optimal   D. Clock
Answer
C. Optimal (OPT / Bélády's Optimal). Replace the page that will not be used for the longest future time. This is provably optimal (minimum page faults) but cannot be implemented in practice since future references are unknown. Used as a benchmark.
8. In RAID 5 with 4 disks each of 500 GB, the usable storage is:
A. 2000 GB   B. 1500 GB   C. 1000 GB   D. 500 GB
Answer
B. 1500 GB. RAID 5 usable = (N−1) × disk_size = (4−1) × 500 = 1500 GB. One disk equivalent is used for distributed parity. The parity is striped across all 4 disks, not on one dedicated disk.
9. Thrashing occurs when:
A. CPU utilization is very high   B. Processes spend more time paging than executing   C. Too few processes are in the system   D. Memory is sufficient for all processes
Answer
B. Thrashing: process spends more time servicing page faults than doing useful computation. CPU utilization drops (processes always blocked on I/O). Paradoxically, increasing multiprogramming makes it worse — fewer frames per process → more faults.
10. The inode in UNIX stores all of the following EXCEPT:
A. File permissions   B. File size   C. File name   D. Block pointers
Answer
C. File name. The inode stores all metadata (permissions, size, timestamps, owner, block pointers) but NOT the file name. The file name is stored in the directory entry, which maps name→inode number. This allows multiple names (hard links) for one inode.

Part II — 2 Mark Questions

11. Processes: P1(AT=0,BT=6), P2(AT=2,BT=4), P3(AT=4,BT=2). Using SRTF, what is the average waiting time?
A. 10/3   B. 3   C. 1   D. 4
Answer
Corrected answer: Gantt chart with tie-breaking: if remaining times are equal, keep the currently running process.
t=0–2: P1 runs (rem=4). t=2: P2 arrives (BT=4), P1_rem=4, equal → continue P1.
t=4: P3 arrives (BT=2), P1_rem=2, equal → continue P1.
Gantt: P1[0–6], P2[6–10], P3[10–12].
CT: P1=6, P2=10, P3=12. TAT: 6, 8, 8. WT: 0, 4, 6.
Avg WT = (0+4+6)/3 = 10/3 ≈ 3.33.
If tie-breaking favors new arrivals: at t=4, P3(2) preempts P1(2).
Gantt: P1[0–4], P3[4–6], P1[6–8], P2[8–12].
CT: P1=8, P2=12, P3=6. TAT: 8, 10, 2. WT: 2, 6, 0. Avg = 8/3 ≈ 2.67.
Key: SRTF answers depend on tie-breaking convention. State your assumption.
12. Reference string: 1,2,3,4,1,2,5 with 3 frames (FIFO). Number of page faults = ?
A. 5   B. 6   C. 7   D. 4
Answer
C. 7. Trace: 1(F→[1]), 2(F→[1,2]), 3(F→[1,2,3]), 4(F→evict1→[2,3,4]), 1(F→evict2→[3,4,1]), 2(F→evict3→[4,1,2]), 5(F→evict4→[1,2,5]). All 7 references miss! Total = 7 page faults.
13. Inode with 10 direct, 1 single-indirect pointer. Block size=1KB, pointer=4B. Number of disk accesses to read the 500th byte of a 300KB file = ?
A. 1   B. 2   C. 3   D. 4
Answer
A. 1. Block size=1KB. 500th byte is in block 0 (bytes 0–1023). Block 0 is a direct pointer (one of 10 direct). Inode has the direct pointer — just follow it → 1 disk access to read the data block. (Inode is assumed already in memory; if not, +1 for inode read.)
14. Which of the following is NOT a necessary condition for deadlock?
A. Mutual Exclusion   B. No Preemption   C. Circular Wait   D. Starvation
Answer
D. Starvation. The four Coffman conditions (necessary and sufficient together): Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait. Starvation is a related but different concept — a process may starve without any deadlock occurring (e.g., priority scheduling without aging).
15. A disk has 7200 RPM. What is the average rotational latency?
A. 4.17 ms   B. 8.33 ms   C. 2.08 ms   D. 16.67 ms
Answer
A. 4.17 ms. Full rotation time = 60,000ms/7200 = 8.33ms. Average rotational latency = half rotation = 8.33/2 = 4.17 ms. Full rotation is the worst case; average is half rotation.
