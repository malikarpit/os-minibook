Unit 1 — OS Intro & Linux Kernel

🎓 University

Short Answer (2 Marks each)

Q1. Define an Operating System. List its four main functions.
An OS is system software that manages computer hardware/software resources and provides common services for programs. Main functions: (1) Process management — create/schedule/terminate processes. (2) Memory management — allocate/deallocate RAM. (3) File system management — organize, store, retrieve data. (4) I/O management — control hardware devices.
Q2. Differentiate between Monolithic and Microkernel architectures.
Monolithic: Entire OS runs in kernel space as one large program. All services (FS, drivers, scheduling) in kernel. Fast (no IPC overhead) but large, less reliable (any bug crashes system). Example: Linux, Unix. Microkernel: Only core functions in kernel (IPC, basic scheduling). Everything else runs as user-space servers. Slow (IPC overhead) but more reliable/secure. Example: Mach, QNX, MINIX.
Q3. What is a system call? Give three examples.
A system call is the programmatic interface between a user process and the OS kernel. When a program needs a privileged operation (disk I/O, creating a process), it invokes a system call, which causes a trap to kernel mode. Examples: (1) fork() — create a child process. (2) read() — read data from a file descriptor. (3) exit() — terminate the current process.
Q4. What is the difference between a process and a program?
A program is a passive entity — a set of instructions stored in a file on disk. A process is an active entity — a program in execution with its own PCB, memory space (stack, heap, code, data), and OS resources. One program can spawn multiple processes (e.g., running two instances of Firefox).

Medium Answer (5 Marks each)

Q5. Explain the five-state process model with a state transition diagram.
States: New (process being created) → Ready (waiting in ready queue for CPU) → Running (executing on CPU) → Waiting/Blocked (waiting for I/O or event) → Terminated (finished). Transitions: New→Ready (admitted), Ready→Running (dispatcher), Running→Ready (preemption/time quantum), Running→Waiting (I/O request), Waiting→Ready (I/O complete), Running→Terminated (exit). Note: process can never go directly from Waiting to Running — must go through Ready first.
Q6. What is the Process Control Block (PCB)? What information does it store?
PCB is a data structure in the OS kernel that represents a process. Contains: Process ID (PID), parent PID (PPID), process state (new/ready/running etc.), Program Counter (address of next instruction), CPU registers (saved on context switch), memory maps (page table, base/limit registers), open file table, scheduling info (priority, queue pointers), I/O status info (devices allocated, pending I/O list). The PCB is the OS's complete record of a process — context switch saves one PCB and restores another.
Q7. Describe the boot process of a Linux system from power-on to login prompt.
1. BIOS/UEFI POST: Hardware self-test, detect devices. 2. Bootloader (GRUB): Loaded from MBR/EFI partition, presents OS menu, loads kernel image. 3. Kernel init: Kernel decompresses itself, initializes hardware drivers, mounts root filesystem. 4. systemd (PID 1): First user-space process, reads units, starts services in dependency order. 5. getty/login: systemd starts getty on terminals, presents login prompt. 6. Shell: After authentication, user's shell starts.

Long Answer (10 Marks each)

Q8. Compare Batch, Multiprogramming, Time-sharing, and Real-time OS with examples.
Batch OS: Jobs submitted as decks, executed without interaction. No user-OS interaction during execution. Efficient CPU use, no interactivity. Examples: IBM mainframes (1960s). Multiprogramming: Multiple jobs in RAM simultaneously. When one does I/O, CPU switches to another. Goal: maximize CPU utilization. No time limits per job. Example: IBM OS/360. Time-sharing (Multitasking): Each user gets small CPU time slices (quantum). Appears interactive — multiple users use simultaneously. Goal: minimize response time. Example: UNIX, Linux. Real-time OS: Strict time constraints. Hard RTOS: deadline miss = system failure (aircraft control). Soft RTOS: deadline miss = degraded quality (video streaming). Examples: VxWorks, FreeRTOS.
⚡ GATE-Style
G1. Which of the following is NOT part of a PCB?
A. Program Counter B. Stack Pointer C. Source code of program D. Open file list
Answer
C. Source code is part of the program (stored on disk), not the process's runtime state. PCB stores runtime information: PC, registers, memory maps, open files, scheduling info.
G2. A process in the "Waiting" state can transition directly to which state?
A. Running B. Terminated C. Ready D. New
Answer
C. Ready. When the I/O or event a process was waiting for completes, it moves to the Ready queue. It cannot go directly to Running — the CPU scheduler must pick it from the ready queue.

Unit 2A — Process Management & Scheduling

🎓 University

Short Answer (2 Marks)

Q1. Define TAT, WT, and Response Time in CPU scheduling.
TAT (Turnaround Time): Time from process submission to completion. TAT = Completion Time − Arrival Time. WT (Waiting Time): Time a process spends in the ready queue. WT = TAT − Burst Time. Response Time: Time from submission until the process first gets CPU. Response Time = First CPU start − Arrival Time. Response time important for interactive systems.
Q2. What is the convoy effect in FCFS scheduling?
Convoy effect occurs when one long CPU-bound process holds the CPU and many short I/O-bound processes queue behind it. The short processes wait a long time for the CPU even though they need very little CPU time. This leads to poor CPU and I/O device utilization. Solution: Use SJF or SRTF which prioritizes shorter bursts.
Q3. Differentiate between preemptive and non-preemptive scheduling.
Non-preemptive: Once CPU is allocated, process runs until it terminates or voluntarily gives up CPU (I/O wait). Simple, no context switch overhead. Examples: FCFS, SJF. Preemptive: OS can forcibly remove CPU from running process (on timer, higher priority arrival). Better responsiveness. Examples: SRTF, Round Robin, Priority (preemptive). Context switch overhead but better for interactive systems.

Numericals (5–10 Marks)

Q4. [10 marks] Processes: P1(AT=0,BT=10), P2(AT=1,BT=3), P3(AT=2,BT=2), P4(AT=3,BT=5). Compute avg TAT and WT for: (a) FCFS (b) SJF (c) SRTF (d) Round Robin (Q=2).
Solution
(a) FCFS (non-preemptive, in arrival order):
Gantt: P1[0-10] P2[10-13] P3[13-15] P4[15-20]
CT: P1=10, P2=13, P3=15, P4=20
TAT: P1=10, P2=12, P3=13, P4=17  → Avg TAT = 52/4 = 13
WT:  P1=0,  P2=9,  P3=11, P4=12  → Avg WT  = 32/4 = 8

(b) SJF (non-preemptive):
At t=0: only P1 → run P1[0-10]
At t=10: P2(3),P3(2),P4(5) → shortest is P3
Gantt: P1[0-10] P3[10-12] P2[12-15] P4[15-20]
CT: P1=10, P2=15, P3=12, P4=20
TAT: P1=10, P2=14, P3=10, P4=17  → Avg TAT = 51/4 = 12.75
WT:  P1=0,  P2=11, P3=8,  P4=12  → Avg WT  = 31/4 = 7.75

(c) SRTF (preemptive SJF):
t=0: P1 runs (only one). t=1: P2(3) < P1_rem(9) → preempt P1, run P2
t=2: P3(2) < P2_rem(2)? Equal → continue P2. t=3: P2(1) < P3(2) → P2
t=4: P2 done. P1_rem(9),P3(2),P4(5) → P3 shortest
t=6: P3 done. P1_rem(9),P4(5) → P4. t=11: P4 done → P1
t=20: P1 done.
Gantt: P2[1-4] P3[4-6] P4[6-11] P1[0-1,11-20]
CT: P1=20,P2=4,P3=6,P4=11
TAT: P1=20,P2=3,P3=4,P4=8   → Avg TAT = 35/4 = 8.75
WT: P1=10,P2=0,P3=2,P4=3   → Avg WT  = 15/4 = 3.75 ← best!

(d) Round Robin Q=2:
Ready queue trace at each quantum...
Gantt: P1[0-2]P2[2-4]P3[4-6]P4[6-8]P1[8-10]P2[10-11]P4[11-13]P1[13-15]P4[15-17]P1[17-19]P1[19-20]
CT: P1=20,P2=11,P3=6,P4=17
TAT: P1=20,P2=10,P3=4,P4=14  → Avg TAT = 48/4 = 12
WT:  P1=10,P2=7, P3=2,P4=9   → Avg WT  = 28/4 = 7
Q5. What is the difference between a thread and a process? What are user-level vs kernel-level threads?
A process has its own address space, file descriptors, and resources. Threads within a process share the same code, data, heap, and files but have separate stacks, registers, and program counters. Threads are lighter (cheaper to create/switch). User-level threads: Managed by user-space library (e.g., GNU Pth, old Java green threads). Kernel sees only one process. Fast context switch but if one thread blocks on I/O, all threads block. Kernel-level threads: OS knows about each thread. True parallelism on multi-core. Slightly more expensive to create/switch. Linux uses 1:1 model (each user thread = one kernel thread via NPTL). Note: POSIX pthreads is an API specification, not inherently user-level. On modern Linux, pthreads = 1:1 kernel-scheduled threads.
⚡ GATE-Style
G1. SJF is optimal in terms of which criterion?
A. Maximum throughput B. Minimum average waiting time C. Minimum response time D. Maximum CPU utilization
Answer
B. Minimum average waiting time (under standard assumptions: known deterministic bursts, single-CPU, no I/O). SJF (non-preemptive) minimizes average WT among all non-preemptive algorithms when burst times are known in advance. SRTF (preemptive SJF) minimizes average WT among ALL algorithms under the same assumptions. Note: In practice, burst times are unpredictable, limiting SJF applicability.
G2. Processes P1(AT=0,BT=5), P2(AT=1,BT=3), P3(AT=2,BT=1). Using SRTF, completion time of P1 = ?
A. 5 B. 9 C. 7 D. 6
Answer
B. 9. t=0:P1 runs. t=1:P2(3)<P1_rem(4)→preempt P1. t=2:P3(1)<P2_rem(2)→preempt P2. t=3:P3 done,P2_rem=2,P1_rem=4→P2. t=5:P2 done,P1_rem=4→P1. t=9:P1 done. CT(P1)=9.

Unit 2B — Concurrency & Synchronization

🎓 University

Short Answer (2 Marks)

Q1. State the three requirements of the critical section problem.
1. Mutual Exclusion: At most one process can be in its critical section at any time. 2. Progress: If no process is in its CS and some want to enter, only those not in their remainder section can participate in the decision — decision cannot be postponed indefinitely. 3. Bounded Waiting: After a process requests entry to its CS, there is a bound on how many times other processes can enter before this one is granted access (prevents starvation).
Q2. What is a race condition? Give an example.
A race condition is a situation where the outcome of a computation depends on the relative timing/interleaving of multiple processes/threads accessing shared data. Example: Two threads both read counter=5, both execute counter++, both write back 6. Result is 6 instead of 7 — one increment is lost. This is a race condition on the counter variable — proper synchronization (mutex/semaphore) prevents it.

Medium Answer (5 Marks)

Q3. Explain Peterson's solution. Does it satisfy all three critical section requirements?
Peterson's solution uses two shared variables: flag[2] (intent to enter) and turn (whose turn to enter). Process Pi: set flag[i]=true, turn=j, then wait while(flag[j]==true AND turn==j). On exit: flag[i]=false. Mutual exclusion: Both cannot enter simultaneously — if both try, turn decides one winner. Progress: If only one wants CS, it enters immediately (other's flag is false). Bounded waiting: After Pi sets turn=j, Pj gets one chance, then Pi enters. All three satisfied! But: Peterson's requires atomic load/store, which modern CPUs may not guarantee without memory barriers.
Q4. Solve the Producer-Consumer problem using semaphores. Show the pseudocode.
Producer-Consumer Pseudocode
// Shared semaphores:
semaphore mutex = 1;   // mutual exclusion for buffer access
semaphore empty = N;   // number of empty slots (initially N)
semaphore full  = 0;   // number of filled slots (initially 0)

// Producer:
do {
    produce_item(item);
    wait(empty);        // wait for an empty slot
    wait(mutex);        // lock buffer
    buffer[in] = item;  // insert item
    in = (in + 1) % N;
    signal(mutex);      // unlock buffer
    signal(full);       // signal one more full slot
} while(true);

// Consumer:
do {
    wait(full);         // wait for a filled slot
    wait(mutex);        // lock buffer
    item = buffer[out]; // remove item
    out = (out + 1) % N;
    signal(mutex);      // unlock buffer
    signal(empty);      // signal one more empty slot
    consume_item(item);
} while(true);

// CRITICAL: wait(mutex) MUST come AFTER wait(empty/full)
// Reversing them causes DEADLOCK!

Long Answer (10 Marks)

Q5. Explain the Dining Philosophers problem. What is the deadlock scenario and how is it solved?
Five philosophers sit at a round table. Each needs 2 chopsticks to eat. There is one chopstick between each pair of philosophers. Problem: If all philosophers simultaneously pick up their left chopstick, all hold one chopstick and wait for the right one — circular wait = deadlock. Solutions: (1) Allow at most 4 philosophers to be seated (asymmetry breaks circular wait). (2) Allow a philosopher to pick up chopsticks only if both are available (atomic pickup). (3) Odd philosophers pick left then right; even pick right then left (breaks symmetry). (4) Use a semaphore/monitor to enforce that only N-1 philosophers can attempt to eat simultaneously. The dining philosophers illustrates deadlock and the need for deadlock avoidance in resource allocation.
⚡ GATE-Style
G1. In the bounded buffer problem, if a producer calls wait(mutex) before wait(empty), what happens when buffer is full?
A. Deadlock B. Busy waiting C. Starvation D. No problem
Answer
A. Deadlock. If the buffer is full and producer holds mutex then waits on empty (which is 0), it blocks. Consumer tries to wait(mutex) but mutex=0 (producer holds it), so consumer blocks too. Neither can proceed — deadlock. Correct order: wait(empty) THEN wait(mutex).
G2. A semaphore S is initialized to 5. After 3 wait() and 1 signal() operations, the value of S is:
A. 3 B. 2 C. 4 D. 1
Answer
A. 3. Starting at 5: wait()→4, wait()→3, wait()→2, signal()→3. Final value = 3.

Unit 2C — Deadlocks

🎓 University

Short Answer (2 Marks)

Q1. State the four Coffman conditions for deadlock.
1. Mutual Exclusion: At least one resource must be held in a non-sharable mode — only one process at a time can use it. 2. Hold and Wait: A process holds at least one resource and is waiting to acquire additional resources held by other processes. 3. No Preemption: Resources cannot be forcibly taken away — a process can only release resources voluntarily. 4. Circular Wait: There must be a circular chain of processes {P0,P1,...,Pn} where each Pi is waiting for a resource held by P(i+1), and Pn is waiting for P0. ALL four conditions must hold simultaneously for deadlock.
Q2. Distinguish between deadlock prevention and deadlock avoidance.
Prevention: Proactively break at least one Coffman condition so deadlock is structurally impossible. Methods: (1) Deny mutual exclusion — use sharable resources. (2) Deny hold-and-wait — request all resources at once. (3) Allow preemption — OS can preempt resources. (4) Deny circular wait — impose total resource ordering. Cons: low resource utilization. Avoidance: Use runtime information (Max matrix) to check if granting a request leaves system in a safe state. Banker's algorithm is the example. Pros: more flexible than prevention. Cons: requires advance declaration of maximum needs.

Numerical (10 Marks)

Q3. [10 marks] 4 processes, 3 resource types (A=9, B=3, C=6). Current state shown. Find: (a) Need matrix, (b) Available, (c) Safe sequence, (d) Can P1 request (1,0,0)?
Banker's Solution
Given:
         Alloc    Max        Need=Max-Alloc    Available?
         A B C    A B C      A B C
P0       0 1 0    7 5 3      7 4 3
P1       2 0 0    3 2 2      1 2 2
P2       3 0 2    9 0 2      6 0 0
P3       2 1 1    4 2 2      2 1 1

(a) Need matrix computed above ✅

(b) Available = Total − Σ Allocation
    A: 9−(0+2+3+2)=2, B:3−(1+0+0+1)=1, C:6−(0+0+2+1)=3
    Available = (2,1,3)

(c) Safety Algorithm, Work=(2,1,3):
    P1: Need=(1,2,2) ≤ (2,1,3)? B:2>1 ✗ — skip
    P3: Need=(2,1,1) ≤ (2,1,3)? ✅ → Work=(2+2,1+1,3+1)=(4,2,4)
    P1: Need=(1,2,2) ≤ (4,2,4)? ✅ → Work=(4+2,2+0,4+0)=(6,2,4)
    P0: Need=(7,4,3) ≤ (6,2,4)? A:7>6 ✗ — skip
    P2: Need=(6,0,0) ≤ (6,2,4)? ✅ → Work=(9,2,6)
    P0: Need=(7,4,3) ≤ (9,2,6)? B:4>2 ✗ — skip!
    
    All remaining allocated? P0 not finished — no safe sequence including P0?
    Re-check: P0 Need=(7,4,3). After P2: Work=(9,2,6). 4>2 still fails.
    No safe sequence exists — the system is in an UNSAFE state!
    (Note: unsafe ≠ deadlocked. Unsafe means deadlock MAY occur
     in some future sequence. The system is not necessarily deadlocked yet.)

(d) P1 requests (1,0,0):
    Check: (1,0,0) ≤ Need[P1]=(1,2,2) ✅
    Check: (1,0,0) ≤ Available=(2,1,3) ✅
    Tentatively grant: Available=(1,1,3), Alloc[P1]=(3,0,0), Need[P1]=(0,2,2)
    Run safety → if already unsafe, deny. Answer: DENY (state already unsafe)
⚡ GATE-Style
G1. For multi-instance resources, a cycle in RAG:
A. Is sufficient for deadlock B. Is necessary but not sufficient for deadlock C. Is necessary and sufficient D. Has no relation to deadlock
Answer
B. For single-instance resources: cycle ↔ deadlock (necessary and sufficient). For multi-instance: cycle is necessary but NOT sufficient. Multiple instances mean a cycle can exist without deadlock if processes can complete without the cycled resources.

Unit 3 — Memory Management

🎓 University

Short Answer (2 Marks)

Q1. Distinguish internal and external fragmentation.
Internal fragmentation: Wasted space INSIDE an allocated block. Occurs when allocation is in fixed-size units larger than the request (e.g., paging — last page may be partially used). External fragmentation: Enough total free memory exists but it's scattered in non-contiguous holes. No single contiguous block is large enough. Occurs in variable-size allocation (contiguous memory schemes). Paging eliminates external fragmentation but causes internal. Segmentation eliminates internal but causes external.
Q2. What is Bélády's Anomaly? Which algorithm suffers from it?
Bélády's Anomaly: increasing the number of page frames causes MORE page faults (counter-intuitive). Only FIFO page replacement algorithm exhibits this. LRU and Optimal algorithms are "stack algorithms" — they are immune to Bélády's anomaly because the set of pages in memory with n frames is always a subset of the set with n+1 frames (stack property). FIFO violates this property.

Numerical (5–10 Marks)

Q3. TLB hit rate = 90%, TLB access time = 10ns, Memory access time = 100ns. Calculate EAT.
EAT = α(t + m) + (1−α)(t + 2m) where α=0.9, t=10ns, m=100ns
EAT = 0.9(10+100) + 0.1(10+200) = 0.9(110) + 0.1(210) = 99 + 21 = 120 ns
Without TLB: 2m = 200ns. With TLB: 120ns → 40% speedup.
Q4. Reference string: 1,2,3,4,1,2,5,1,2,3,4,5 with 3 frames. Find page faults using LRU and FIFO.
Page Fault Trace
Ref:  1  2  3  4  1  2  5  1  2  3  4  5

FIFO (evict oldest): Faults = 9
Frame state at each reference:
[1]→[1,2]→[1,2,3]→[4,2,3]→[4,1,3]→[4,1,2]→[5,1,2]→hit→hit→[5,3,2]→[5,3,4]→[5,3,4] wait re-trace:
Fault?F   F    F     F     F     F     F    -   -    F     F     F  = 9 faults

LRU (evict least recently used): Faults = 10
Ref:  1  2  3  4  1  2  5  1  2  3  4  5
Fault?F  F  F  F  F  F  F  -  -  F  F  F = 10 faults
(At ref=4: LRU=1, replace; at ref=1: LRU=2, replace; etc.)

Optimal = 7 faults (minimum possible)
Q5. What is thrashing? How does the working set model prevent it?
Thrashing occurs when a process spends more time servicing page faults than executing (CPU utilization drops sharply despite high multiprogramming). Cause: too many processes competing for too few frames — each process's working set exceeds its allocated frames. Working Set Model: Track the set of pages a process has used in the last Δ time units (working set). Sum of all working sets = total frames needed. If sum > available frames, suspend some processes (reduce multiprogramming) until enough frames are available. This prevents thrashing by ensuring each process has enough frames for its current locality.
⚡ GATE-Style
G1. A virtual address space of 32 bits with 4KB pages. Page table entry = 4 bytes. Size of page table = ?
A. 4 MB B. 1 MB C. 16 MB D. 4 KB
Answer
A. 4 MB. Number of pages = 2³²/2¹² = 2²⁰ = 1M entries. Size = 1M × 4 bytes = 4 MB. This is why hierarchical page tables exist — 4MB per process is too expensive.
G2. Page fault rate p=0.001, memory access = 200ns, page fault service = 8ms. EAT = ?
A. 200.2 ns B. 8.2 µs C. 8.2 ms D. 8199.8 ns
Answer
D. 8199.8 ns. EAT = (1-p)×m + p×s = (0.999)(200ns) + (0.001)(8,000,000ns) = 199.8 + 8000 = 8199.8 ns ≈ 8.2 µs. Even 0.1% fault rate causes 41× slowdown vs no faults (200ns)!

Unit 4 — File Systems & Storage

🎓 University

Short Answer (2 Marks)

Q1. What information is stored in an inode? What is NOT stored?
Stored in inode: File type, permissions (rwxrwxrwx), owner (uid/gid), file size, timestamps (atime/mtime/ctime), hard link count, block pointers (12 direct + 1 single + 1 double + 1 triple indirect). NOT stored in inode: File name! The file name is stored in the directory entry, which maps (name → inode number). This allows hard links — multiple directory entries pointing to the same inode.
Q2. Compare hard links and symbolic (soft) links.
Hard link: Another directory entry pointing to the same inode. Both names point to same data. Deleting one doesn't delete the other (inode remains until link count = 0). Cannot cross filesystem boundaries. Cannot link directories. Symbolic link: A file whose content is the path to the target. Acts like a shortcut. Can cross filesystems. Can link directories. If target is deleted, symlink becomes dangling (broken).

Numerical (5–10 Marks)

Q3. Disk requests: 98,183,37,122,14,124,65,67. Head at 53. Calculate total head movement for FCFS, SSTF, and SCAN (moving toward higher).
Disk Scheduling Solution
FCFS: 53→98→183→37→122→14→124→65→67
Movement: 45+85+146+85+108+110+59+2 = 640 cylinders

SSTF (always nearest):
From 53: nearest=65(12)→67(2)→37(30)→14(23)→98(84)→122(24)→124(2)→183(59)
Total = 12+2+30+23+84+24+2+59 = 236 cylinders

SCAN (moving right, toward higher):
Service right of 53: 65,67,98,122,124,183 then reverse to 37,14
53→65(12)→67(2)→98(31)→122(24)→124(2)→183(59)→199(16)→37(162)→14(23)
Total = 12+2+31+24+2+59+16+162+23 = 331 cylinders
(Formula: (199-53) + (199-14) = 146+185 = 331)
Q4. Inode has 12 direct, 1 single-indirect pointer. Block size = 1 KB, pointer = 4 bytes. What is the max file size?
Pointers per block = 1024/4 = 256. Direct blocks = 12 × 1KB = 12 KB. Single indirect = 256 × 1KB = 256 KB. Max file size = 12 + 256 = 268 KB. With double indirect: 268 + 256² × 1KB = 268 + 64MB ≈ 64 MB. With triple: ≈ 16 GB.
Q5. [10 marks] Explain RAID 0, 1, and 5. A server needs 200 GB usable storage with fault tolerance for 1 disk failure. Using 100 GB disks, how many are needed for RAID 5?
RAID 5 usable = (N−1) × disk_size. Need 200 GB: (N−1) × 100 = 200 → N−1 = 2 → N = 3 disks. Total = 300 GB physical, 200 GB usable, 100 GB for distributed parity. Survives 1 disk failure. For 2-disk failure tolerance, use RAID 6: need (N−2) × 100 = 200 → N = 4 disks (400 GB total, 200 GB usable).
⚡ GATE-Style
G1. Which file allocation method supports direct access in O(1) and has no external fragmentation?
A. Contiguous B. Linked C. Indexed D. Both A and C
Answer
C. Indexed allocation only. Contiguous allocation gives O(1) direct access (start+i) but suffers from external fragmentation. Linked allocation avoids external fragmentation but does NOT support O(1) direct access (must traverse chain). Indexed allocation (inode) supports direct access via index block AND has no external fragmentation. Only C satisfies both constraints.
G2. A bitmap for free space management of a 1 GB disk with 512-byte blocks needs how much space?
A. 256 KB B. 512 KB C. 1 MB D. 128 KB
Answer
A. 256 KB. Number of blocks = 1 GB / 512 B = 2³⁰/2⁹ = 2²¹ = 2M blocks. Bitmap size = 2M bits = 2M/8 bytes = 256 KB.
